Contexte biologique

Du cancer au vaccin personnalisé

Le système immunitaire en 1 minute

Immunité innée

  • Réponse rapide, non-spécifique
  • Macrophages, neutrophiles, cellules NK
  • Première ligne de défense
  • Pas de mémoire

Immunité adaptative

  • Réponse spécifique et ciblée
  • Lymphocytes T (CD4+, CD8+) et B
  • Reconnaissance d'antigènes précis
  • Mémoire → vaccins

Dans ce projet, on s'intéresse aux lymphocytes T CD8+ (immunité adaptative) — les "tueurs" qui éliminent les cellules anormales.

Lymphocytes T CD8+ : les tueurs

  • Chaque cellule nucléée expose des fragments de ses protéines à sa surface
  • Ces fragments (peptides) sont présentés par les molécules HLA classe I
  • Les T CD8+ patrouillent et inspectent ces peptides
  • Si un peptide est reconnu comme étranger → la cellule est détruite
Analogie : chaque cellule montre sa "carte d'identité" (peptides du soi). Si la carte montre quelque chose d'anormal (peptide mutant) → le T CD8+ élimine la cellule.
Cellule normale
peptides du soi → ignorée

Cellule tumorale
peptide mutant → détruite !

Cellule infectée
peptide viral → détruite !

À chaque étape, des peptides sont éliminés

  • Le protéasome ne coupe pas n'importe où
  • TAP ne transporte pas tous les peptides
  • ERAP peut trop raccourcir le peptide
  • Le HLA n'accepte que certaines séquences

Lien avec vos modules

  • Dept B modélise les étapes 2, 3, 4
  • Dept C modélise l'étape 5
  • Dept A évalue le peptide globalement
  • Dept D vérifie que c'est bien du "non-soi"

Mutations somatiques

  • Les cellules tumorales accumulent des mutations somatiques
  • Certaines modifient la séquence protéique
  • Ces protéines mutées sont dégradées en peptides (8-11 acides aminés)
  • Certains peptides sont présentés à la surface cellulaire

Présentation HLA classe I

  • Les molécules HLA classe I présentent des peptides à la surface de toutes les cellules nucléées
  • Les lymphocytes T CD8+ inspectent ces peptides
  • Si le peptide est reconnu comme non-soi → réponse cytotoxique
  • Chaque individu possède un set d'allèles HLA différent (ex: A*02:01, B*07:02...)

Sillon HLA-I avec positions d'ancrage P2 et P9

Néo-épitopes : cibles vaccinales

  • Un néo-épitope = peptide mutant présenté par HLA et reconnu comme non-soi
  • Cible idéale pour un vaccin thérapeutique personnalisé (ARNm)
  • Le défi : parmi des milliers de mutations, lesquelles sont les bonnes cibles ?

Un bon candidat doit satisfaire plusieurs critères simultanés :

Se fixe au HLA Est présenté Diffère du soi Physico-chimie OK

L'API : une seule fonction

Chaque module expose une seule fonction publique :

def get_score(candidate: Candidate) -> tuple[str, float]:
    """
    Parameters: candidate — un objet Candidate (peptide_wt, peptide_mut, ...)
    Returns:    ("nom_du_score", valeur_numérique)
    """
    # ... votre logique ici ...
    return ("hydrophobicity", 0.72)

L'objet Candidate

  • candidate_id — identifiant
  • peptide_wt — séquence wild-type
  • peptide_mut — séquence mutante
  • mut_pos_1based — position de la mutation

Règles

  • Retourner un tuple de 2 éléments
  • Le nom doit être unique et descriptif
  • La valeur doit être un float fini
  • Pas de NaN, pas d'infini

Départements & modules

4 axes de scoring, 15 modules

4 départements complémentaires

A — Propriétés physico-chimiques

Le peptide a-t-il des propriétés raisonnables ?

Hydrophobicité, charge, complexité, delta WT/MUT

B — Présentation cellulaire

Le peptide peut-il arriver jusqu'au HLA ?

Protéasome, TAP, ERAP, sanity checks

C — Liaison HLA (coeur du projet)

Le peptide se fixe-t-il bien dans le sillon HLA ?

Ancres P2/P9, ancres secondaires, delta binding

D — Sécurité & soi

Le peptide est-il vraiment "non-soi" ?

Self-similarity, mutation dans la fenêtre

Aucun module seul ne suffit — c'est la combinaison qui fait un bon pipeline.

Département A — Propriétés physico-chimiques

Le peptide a-t-il un profil physico-chimique compatible avec une présentation immune ?

ModuleDescriptionIdée
A1Hydrophobicité (Kyte-Doolittle)Pénaliser les extrêmes : trop hydrophobe = pas soluble, trop hydrophile = pas de liaison HLA
A2Charge nette / pILes peptides très chargés sont mal présentés
A3Complexité / entropieUn peptide répétitif (AAAAA) est suspect
A4Delta WT vs MUTSi la mutation ne change rien physico-chimiquement → mauvais signe
Piège typique : CAND_11 (ILVMILMVL) — score d'hydrophobicité excellent mais c'est un MAUVAIS candidat (impossible à présenter).

Département B — Présentation cellulaire

Le peptide peut-il survivre au trajet intracellulaire jusqu'au HLA ?

ModuleDescriptionIdée
B1Protéasome (C-terminal)Le protéasome coupe préférentiellement après certains AA
B2TAP (transport)Le transporteur TAP favorise certains peptides
B3ERAP (N-terminal)ERAP peut raccourcir le peptide — certains N-ter résistent mieux
B4Sanity checksVérifications de base : longueur, caractères valides, cohérence
Piège typique : CAND_16 (ALXDF*NQV) — contient des caractères invalides (X, *). Seul B4 le détecte.

Département C — Liaison HLA

Le peptide se fixe-t-il bien dans le sillon de liaison HLA ? C'est le coeur du projet.

ModuleDescriptionIdée
C1Ancre P2 (PSSM)Position 2 : L ou M sont idéaux pour HLA-A*02:01
C2Ancre P9 / PΩDernière position : V ou L sont idéaux
C3Ancres secondairesLes autres positions contribuent aussi (mais moins)
C4Delta binding WT/MUTLa mutation améliore-t-elle la liaison HLA ?
Outil clé : la matrice PSSM (hla_pssm_A0201.csv) — scores log-odds par acide aminé et par position.

Département D — Sécurité & soi

Le peptide est-il vraiment différent des peptides humains normaux ?

ModuleDescriptionIdée
D1Self-similarity exacteLe peptide mutant existe-t-il dans le protéome humain ?
D2Self-similarity approx.Distance de Hamming avec le corpus humain
D3Mutation dans la fenêtreLa mutation est-elle vraiment dans le peptide analysé ?
Pièges typiques :
CAND_14 — peptide identique à un peptide humain connu (self-match exact, D1).
CAND_06 — WT == MUT, la mutation n'existe pas (D3). CAND_18 — mut_pos hors fenêtre (D3).

Pourquoi on a besoin de TOUS les modules

Chaque module seul a des angles morts :

Le workflow Git

1 git clone — Récupérer le projet
2 git checkout -b groupe-XX — Créer votre branche

Installation

# 1. Cloner le dépôt
git clone https://github.com/open-neovax-course/open-neovax.git
cd open-neovax

# 2. Créer l'environnement virtuel
python3 -m venv .venv
source .venv/bin/activate    # Linux / macOS
# .venv\Scripts\activate     # Windows

# 3. Installer les dépendances
pip install -r requirements.txt

# 4. Vérifier que tout marche
pytest tests/ -v             # → 22 passed
ruff check .                 # → All checks passed
black --check .              # → All done

# 5. Lancer l'interface
streamlit run app.py
