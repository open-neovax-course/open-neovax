La PSSM — hla_pssm_A0201.csv

Matrice Position-Specific Scoring Matrix pour HLA-A*02:01 :

AAP1P2P3...P9
L0.5+2.00.1...+1.5
D-1.0-2.5-0.5...-2.0
V0.3+1.00.2...+2.0
..................
  • Positif = AA favorable à cette position
  • Négatif = AA défavorable
  • P2 et P9 = positions d'ancrage (les plus importantes)

import pandas as pd

Chargement robuste

Ne jamais crasher, même si le fichier n'est pas là

Le pattern DATA_DIR

Bien : chemin relatif au module 
DATA_DIR = Path(__file__).resolve() \
    .parent.parent / "data"

path = DATA_DIR / "hla_pssm_A0201.csv"
  • Fonctionne depuis n'importe quel dossier
  • Fonctionne sur le PC de n'importe qui
  • Fonctionne dans la CI GitHub
Mal : chemin absolu 
path = "/home/thomas/open-neovax/data/hla_pssm_A0201.csv"
  • Fonctionne uniquement sur VOTRE machine
  • Crashe dans la CI
  • Crashe chez votre binôme
Mal aussi : chemin relatif au CWD 
path = "data/hla_pssm_A0201.csv"

Ne fonctionne que si on lance depuis la racine du projet.

Gérer le fichier absent

Votre module ne doit jamais crasher, même si le fichier de données manque.

Pattern recommandé : 
_PSSM = None

def _load_pssm():
    global _PSSM
    if _PSSM is not None:
        return _PSSM
    path = DATA_DIR / "hla_pssm_A0201.csv"
    if not path.exists():
        return None  # fichier absent
    df = pd.read_csv(path)
    _PSSM = df.set_index("aa")
    return _PSSM

def get_score(candidate):
    pssm = _load_pssm()
    if pssm is None:
        return ("C_hla_anchor_p2", 0.0)
    # ... calcul normal

Pourquoi ce pattern ?

  • Lazy loading : on ne charge qu'au premier appel
  • Cache : la variable globale _PSSM évite de relire le CSV 18 fois (1 par candidat)
  • Graceful fallback : si le fichier manque, on retourne un score neutre (0.0) au lieu de crasher
Même logique pour le corpus self :
Charger le set une seule fois, le stocker dans une variable globale, retourner 0.0 si le fichier est absent.

Dept B — Processing & présentation

Tables de préférence pour le clivage et le transport. Constantes dans le code ou petit CSV.

ModuleCe que vous codez aujourd'hui
B1Table de clivage protéasomal : score du dernier (et avant-dernier) AA
B2Score TAP : basé sur les 3 premiers AA du peptide
B3ERAP : résistance au trimming N-terminal (1er AA)
B4Sanity : longueur 8-11, AA valides, WT != MUT
Exemple B1 — table de clivage : 
# AA favorables en C-terminal
# pour le clivage protéasomal
CTERM_PREFERENCE = {
    "L": 1.0, "A": 0.8, "V": 0.7,
    "I": 0.7, "M": 0.6, "F": 0.5,
    "K": 0.3, "R": 0.2,
    "D": -0.5, "E": -0.5,
    "P": -1.0,
}

def get_score(candidate):
    pep = candidate.peptide_mut
    last_aa = pep[-1]
    score = CTERM_PREFERENCE.get(
        last_aa, 0.0)
    return ("B_proteasome_cterm", score)

D2 — Distance de Hamming : 
def _hamming(a, b):
    """Nombre de positions différentes."""
    return sum(c1 != c2 for c1, c2 in zip(a, b))

def get_score(candidate):
    pep = candidate.peptide_mut
    corpus = _load_self_peptides()
    if corpus is None:
        return ("D_self_approx_match", 0.0)

    # Distance min avec le corpus
    same_len = [p for p in corpus
                if len(p) == len(pep)]
    if not same_len:
        return ("D_self_approx_match", 0.0)

    min_dist = min(_hamming(pep, p)
                   for p in same_len)
    return ("D_self_approx_match",
            float(min_dist))

Test 2 : candidat connu 
def test_known_candidate_score():
    """Score on a known candidate is
    in the expected range."""
    # CAND_01 : peptide_mut = "SLMAFTIAV"
    # P2 = L → attendu : score élevé
    cand = _make_candidate(
        peptide_mut="SLMAFTIAV"
    )
    name, value = get_score(cand)
    assert value > 0  # L@P2 = favorable
