Gérer les cas limites

Crashe sur CAND_16 : 
def get_score(candidate):
    pep = candidate.peptide_mut
    return ("score", KD[pep[0]])
    # KeyError si pep[0] = "X"
Crashe sur peptide vide : 
def get_score(candidate):
    pep = candidate.peptide_mut
    avg = sum(...) / len(pep)
    # ZeroDivisionError
Robuste : 
def get_score(candidate):
    pep = candidate.peptide_mut
    if not pep or len(pep) < 8:
        return (SCORE_NAME, 0.0)

    total = sum(
        KD.get(aa, 0.0)
        for aa in pep.upper()
    )
    return (SCORE_NAME, total / len(pep))

.get(aa, 0.0) = jamais de KeyError.
Check longueur = jamais de ZeroDivisionError.

Robustesse & tests

3 tests minimum : nominal, edge case, invalide

Les 3 tests que chaque module doit avoir

1. Nominal

Un candidat standard retourne un score raisonnable

def test_nominal():
    name, val = get_score(
        make_cand("SLMAFTIAV"))
    assert isinstance(val, float)
    assert name == SCORE_NAME

2. Edge case

Un candidat problématique ne crashe pas

def test_invalid_aa():
    name, val = get_score(
        make_cand("ALXDF*NQV"))
    assert isinstance(val, float)
    # pas de crash !

3. Invalide

Un input vide/absent retourne un score neutre

def test_empty():
    name, val = get_score(
        make_cand(""))
    assert isinstance(val, float)
    # score neutre, pas crash

Chaque module doit documenter son hypothèse et ses limites :

"""Module A1 — Hydrophobicity (Kyte-Doolittle).

Hypothesis:
    Neo-epitopes need balanced hydrophobicity for efficient
    processing and HLA presentation. Extreme values are penalized.

Computation:
    Position-weighted Kyte-Doolittle average.
    Anchor positions (P2, P9) favor hydrophobic residues.
    Exposed positions (P4-P7) penalize excess hydrophobicity.

Score interpretation:
    High  = favorable hydrophobic profile
    Low   = extreme or unbalanced hydrophobicity

Limitations:
    - Single-scale approximation
    - Does not account for peptide-HLA specific interactions
    - Position weights are heuristic, not data-driven
"""

Le problème : la moyenne simple

Actuellement, le score final est :

total = (score_A1 + score_A2 + score_B1
        + score_C1 + score_C2 + score_D1
        + ...) / N

Chaque module compte autant. Mais est-ce que la charge nette (A2) est aussi prédictive que l'ancre P2 (C1) ?

Probablement pas.

La question centrale

Peut-on faire mieux qu'une moyenne ?

Peut-on apprendre les bons poids à partir des données ?

C'est un problème de machine learning.

Cross-validation

On ne teste jamais sur les données d'entraînement. La cross-validation découpe les données en K plis :

Fold 1 Train sur 80% → Test sur 20%
Fold 2 Train sur 80% → Test sur 20%
Fold 3 Train sur 80% → Test sur 20%
Fold 4 Train sur 80% → Test sur 20%
Fold 5 Train sur 80% → Test sur 20%

Résultat = moyenne des 5 scores.

from sklearn.model_selection import (
    cross_val_score,
)
from sklearn.ensemble import (
    RandomForestClassifier,
)

model = RandomForestClassifier(
    n_estimators=100,
    max_depth=5,
)

scores = cross_val_score(
    model, X, y, cv=5,
    scoring="accuracy",
)

print(f"Accuracy: {scores.mean():.3f}"
      f" (+/- {scores.std():.3f})")
